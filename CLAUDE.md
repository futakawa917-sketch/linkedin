# LinkedIn BtoB集客自動化システム

## プロジェクト概要
SNS HACKのLinkedIn BtoB集客を自動化するシステム。
月間商談目標から必要行動量を逆算し、ターゲットリスト管理・投稿AI生成・日次リマインドを自動化する。

## 2つのアカウント運用
- **個人アカウント**: 営業の主戦場。専門性・事例でリード獲得→DM→商談
- **企業アカウント**: ブランディングの看板。個人投稿の再利用＋企業視点の発信

## LinkedIn規約遵守
- DM送信・つながり申請の自動送信は**禁止**
- AIは文面生成のみ。送信は必ず手動
- コメントでの接点づくりも手動

## ファネル構造
```
ターゲットリスト → コメント → つながり申請 → 承認 → DM → 商談
```

### ステータスフロー
未アプローチ → コメント済 → 申請済 → 承認済 → DM済 → アポ獲得 → 商談済

### タイミングルール
| ステータス | 次のアクション | タイミング |
|---|---|---|
| 未アプローチ | 投稿にコメント | リスト追加から3日以内 |
| コメント済 | つながり申請 | コメント後1-2日 |
| 申請済 | 待機 | 7日後に再確認リマインド |
| 承認済 | DM送信 | 承認後2-3日以内 |
| DM済 | フォローアップ | 5日後にリマインド |

## 投稿設計
- 週3本（月/水/金）
- 投稿タイプ比率: 専門性40% / 事例30% / エンゲージメント20% / CTA10%

### 投稿タイプ
1. **専門性**: SNS運用のノウハウ・業界知見を発信。信頼構築
2. **事例**: クライアントの成功事例（匿名）。社会的証明
3. **エンゲージメント**: 質問・議論喚起。リーチ拡大
4. **CTA**: 「DMください」系。月1-2回に抑える

## ターゲット
- 業界: IT・SaaS、人材、コンサル、マーケティング
- 企業規模: 10〜300名
- 決裁者: 代表取締役、取締役、マーケ責任者、事業部長
- ペイン: SNS集客の始め方がわからない／採用コスト高い／広告費高騰／自社運用で成果出ない

## 技術スタック
- GitHub Actions（スケジュール実行、PC起動不要）
- Claude Sonnet 4.6 API（投稿生成、分析）
- JSON + Markdown（データストア、DB不要）

## ディレクトリ構成
- `config/` — 設定ファイル、プロンプト
- `scripts/` — Python自動化スクリプト
- `data/` — KPI、CRM、分析データ
- `posts/` — 投稿（drafts→approved→posted）
- `knowledge/` — ナレッジベース
- `output/` — 日次アクションリスト、レポート

## 手動作業の時間配分（1日30分）
- コメント: 15分（14件目安）
- つながり申請: 5分（7件）
- DM送信: 5分（2件）
- 投稿確認・編集: 5分
